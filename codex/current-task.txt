## 修正タスク: 為替レート計算バグ修正（残りのファイル）

前回 enrichment-service.ts を修正済み。同じバグが以下のファイルにも存在する。

### 問題のあるファイル
`exchangeRate` はDB上「JPY → USD」（例: 0.0064）で保存されている。
`costJpy / exchangeRate` は逆の結果を返すため `costJpy * exchangeRate` に修正が必要。

1. `apps/worker/src/processors/ebay-publish.ts` 680行目付近:
   ```
   const costUsd = costJpy / exchangeRate;
   ```
   → `costJpy * exchangeRate` に修正

2. `apps/worker/src/lib/ai-service.ts` 528行目と1349行目付近:
   ```
   const costUsd = costJpy / exchangeRate;
   ```
   → 両方とも `costJpy * exchangeRate` に修正

3. `apps/worker/src/lib/price-adjuster.ts` 68-71行目付近:
   ```
   // sellingPriceUsd = sellingPriceJpy / exchangeRate
   const sellingPriceUsd = sellingPriceJpy / exchangeRate;
   ```
   → コメントと実際のコードの両方を修正:
   ```
   // sellingPriceUsd = sellingPriceJpy * exchangeRate
   const sellingPriceUsd = sellingPriceJpy * exchangeRate;
   ```

4. `apps/worker/src/lib/enrichment-service.ts` 445行目:
   ```
   return rate?.rate || 150; // デフォルト150円/USD
   ```
   → DBが JPY→USD の直接レート（0.0064等）を返すため、デフォルトも同じ形式にする:
   ```
   return rate?.rate || 0.0067; // デフォルト1/150（JPY→USD直接レート）
   ```

### 注意
- 各ファイルで `exchangeRate` の取得元を確認し、JPY→USD の直接レートであることを確認してから修正
- price-calculator.ts は既に正しい（`sourcePrice * exchangeRate` を使用）ので触らない
- テストの期待値も必要なら修正
