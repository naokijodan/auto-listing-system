## タスク: Chrome拡張機能メルカリセレクタ修正

### Working directory
~/Desktop/rakuda/

### 問題
Playwright検証でメルカリの一部セレクタが変更されていることが判明:
- `[data-testid="seller-name"]` → 存在しない。正しくは `[data-testid="seller-link"]`
- `[data-testid="category"]` → 存在しない。正しくは `[data-testid="item-detail-category"]`

### 修正箇所

#### 1. extensions/chrome/background.js の extractProductInfo 関数（メルカリ部分）
- 305行目付近の `if (url.includes('mercari.com'))` ブロック内

変更1: sellerName のセレクタ（322行目付近）
```
// 変更前:
const sellerName = document.querySelector('[data-testid="seller-name"]')?.textContent?.trim();

// 変更後:
const sellerName = document.querySelector('[data-testid="seller-link"]')?.textContent?.trim();
```

変更2: category のセレクタ（325行目付近）
```
// 変更前:
const category = document.querySelector('[data-testid="category"]')?.textContent?.trim();

// 変更後:
const category = document.querySelector('[data-testid="item-detail-category"]')?.textContent?.trim();
```

#### 2. extensions/chrome/popup.js のメルカリ部分にも同様のセレクタがあれば修正
- popup.js 内のメルカリセクションで seller-name や category のセレクタがあれば同様に修正

### 注意事項
- ヤフオクとAmazon JPのセレクタは変更しない
- content.js は変更しない
- ファイルの他の部分は一切変更しない
- 既存のコード構造・スタイルに合わせる
