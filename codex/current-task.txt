## Phase 4B: 旧 calculatePrice → 新 PricingPipeline への移行

### 作業ディレクトリ
~/Desktop/rakuda/

### 対象ファイル
1. `apps/worker/src/processors/publish.ts` - 修正
2. `apps/worker/src/processors/price-sync.ts` - 修正
3. `apps/worker/src/lib/price-sync.ts` - 修正

### 概要
旧 `price-calculator.ts` の `calculatePrice()` を使っている3ファイルを、新 `PricingPipeline` に移行する。

### 新パイプラインの使い方
```typescript
import { PricingPipeline } from '../lib/pricing';  // または './pricing'
import { Marketplace } from '@rakuda/database';

const pipeline = new PricingPipeline();
const result = await pipeline.calculate({
  sourcePrice: product.price,      // 仕入価格（円）
  weight: product.weight || 200,   // 重量（g）
  marketplace: Marketplace.JOOM,   // or Marketplace.EBAY
  category: product.category,
  pricingMode: 'DETAILED',         // 新規追加（必須）
  profitMode: 'RATE',              // 新規追加（必須）
});

// 出力マッピング:
// 旧 priceResult.listingPrice → 新 result.finalPrice
// 旧 priceResult.shippingCost → 新 result.breakdown.shippingCostUsd
```

### Marketplace import について
- `apps/worker/src/processors/price-sync.ts` は既に `import { Marketplace } from '@prisma/client';` がある（行3）。しかし、RAKUDA のルールでは `@rakuda/database` からインポートするのが正しい。`@prisma/client` のままでもコンパイルは通るが、もし `@rakuda/database` に統一したい場合は変更する。ただし、既存コードの import を壊さないように注意。今回は既存の import はそのまま残す。
- `apps/worker/src/processors/publish.ts` で Marketplace が import されていなければ追加: `import { Marketplace } from '@rakuda/database';`
- `apps/worker/src/lib/price-sync.ts` に `import { Marketplace } from '@rakuda/database';` を追加
