## タスク: Phase 1 - APIキー設定UI + SystemSettingへの保存

### Working directory
~/Desktop/rakuda/

### 概要
設定画面（/settings）のシステムタブにAPIキー管理セクションを追加し、
OpenAI APIキーをWeb UIから設定できるようにする。

### 背景
- `SystemSetting`テーブルは既にある（packages/database/prisma/schema.prisma:1947）
- `isSecret`フラグ、`SECRET` valueType、バージョン管理、SettingHistory付き
- API: apps/api/src/routes/system-settings.ts に GET/PUT 済み（マスキング対応済み）
- UI: apps/web/src/app/settings/page.tsx の `SystemSettings` コンポーネント（1320行目〜）は現在ハードコード表示のみ

#### 2. UI: SystemSettingsコンポーネントにAPIキー管理セクションを追加

ファイル: `apps/web/src/app/settings/page.tsx`

`SystemSettings`関数コンポーネント（1320行目〜）を改修:

- 既存のスケジューラー・データベースカードはそのまま残す
- 新しい「API連携設定」カードを**先頭に**追加
- 以下のUI要素を含む:
  - OpenAI API Key入力欄（type="password"）
    - 保存済みの場合はマスク表示（「sk-xxxx...xxxx」形式、先頭4文字+末尾4文字）
    - 空欄で保存すると既存値を維持（上書きしない）
    - 保存ボタン
    - 接続テスト（Verify）ボタン → `/api/system-settings/openai_api_key/verify` を呼ぶ
  - OpenAI モデル選択（ドロップダウン: gpt-4o, gpt-4o-mini, gpt-4o-2024-11-20）
  - ステータス表示（設定済み/未設定）

- データ取得: `GET /api/system-settings/category/INTEGRATION`
- 保存: `PUT /api/system-settings/{key}` (既存API使用)
- マスク表示ロジック: APIから返る`********`をそのまま表示するのではなく、
  「設定済み」のステータスバッジを表示。入力欄はプレースホルダーで「新しいキーを入力...」

#### 4. translator.tsの改修: 環境変数→DB読み込みに変更

ファイル: `packages/enrichment/src/translator.ts`

`getOpenAIClient()`を以下のように変更:
- まずDBの`SystemSetting`から`openai_api_key`を取得
- 未設定の場合は環境変数`OPENAI_API_KEY`にフォールバック
- どちらもない場合はnullを返す（現在と同じ挙動）

```typescript
import { prisma } from '@rakuda/database';

async function getApiKeyFromDB(): Promise<string | null> {
  try {
    const setting = await prisma.systemSetting.findUnique({
      where: { key: 'openai_api_key' },
    });
    return setting?.value || null;
  } catch {
    return null;
  }
}

// getOpenAIClient()を非同期に変更
async function getOpenAIClient(): Promise<OpenAI | null> {
  if (!openaiInitialized) {
    openaiInitialized = true;
    const apiKey = await getApiKeyFromDB() || process.env.OPENAI_API_KEY;
    if (apiKey) {
      openai = new OpenAI({ apiKey });
    }
  }
  return openai;
}
```

注意: `getOpenAIClient()`を非同期にするため、呼び出し元の`enrichProduct()`と
`translateToEnglish()`も`await getOpenAIClient()`に変更する。
既存のフォールバック処理（`createFallbackResult`）はそのまま維持。
