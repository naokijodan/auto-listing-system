## タスク: サーキットブレーカーリセットAPIを修正し、スクレイピング用サーキットブレーカーもリセットできるようにする

### 対象ファイル
`apps/api/src/routes/admin.ts`

### 問題
現在の `POST /api/admin/performance/circuit-breaker/:name/reset` エンドポイントは、レガシーのサーキットブレーカー（`rakuda:circuit:${name}`）のみリセットする。
スクレイピング用サーキットブレーカー（`rakuda:scrape:circuit:${name}` と `rakuda:scrape:circuit:${name}:half_open_token`）は別のRedisキーを使っているため、リセットされない。

### 修正内容
`apps/api/src/routes/admin.ts` の1098行目付近にある `/performance/circuit-breaker/:name/reset` エンドポイントを修正し、以下の3つのRedisキーを全てリセットするようにする：

1. `rakuda:circuit:${name}` — レガシーサーキットブレーカー（既存のhsetをそのまま維持）
2. `rakuda:scrape:circuit:${name}` — スクレイピング用サーキットブレーカー（DELで削除）
3. `rakuda:scrape:circuit:${name}:half_open_token` — HALF_OPENガードトークン（DELで削除）

### 具体的な変更

```typescript
// 既存の1098-1121行を以下に置き換え
router.post('/performance/circuit-breaker/:name/reset', async (req: Request, res: Response, next: NextFunction) => {
  try {
    const { name } = req.params;

    // レガシーサーキットブレーカーをリセット
    const legacyKey = `rakuda:circuit:${name}`;
    await redis.hset(legacyKey, {
      state: 'CLOSED',
      failureCount: '0',
      lastStateChange: new Date().toISOString(),
    });

    // スクレイピング用サーキットブレーカーをリセット
    const scrapeCircuitKey = `rakuda:scrape:circuit:${name}`;
    const scrapeHalfOpenKey = `rakuda:scrape:circuit:${name}:half_open_token`;
    await redis.del(scrapeCircuitKey);
    await redis.del(scrapeHalfOpenKey);

    log.info({
      type: 'circuit_breaker_manual_reset',
      name,
      keysReset: [legacyKey, scrapeCircuitKey, scrapeHalfOpenKey],
    });

    res.json({
      success: true,
      message: `Circuit breaker "${name}" has been reset to CLOSED (legacy + scrape)`,
    });
  } catch (error) {
    next(error);
  }
});
```

### 注意事項
- 既存のレガシーリセット処理はそのまま維持すること
- `redis` は既にファイル内でimport済み
- TypeScriptの構文を壊さないこと
- 既存のテストに影響しないこと
