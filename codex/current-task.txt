# タスク: joom-publish-service.ts のTypeScriptコンパイルエラー修正

## 問題
apps/worker/src/lib/joom-publish-service.ts の publishToJoom() メソッドで、
行408で listing を再代入した後、TypeScriptがlistingをnull可能と判定している。
行425以降で listing.product.title 等のアクセスが TS18047 エラーになる。

## エラー箇所
行408: listing = await prisma.listing.findUnique(...)  ← ここで listing が null | Listing になる
行425: listing.product.title ← TS18047: 'listing' is possibly 'null'
（以降、行427, 446, 455, 458, 462, 464, 497, 520 も同様）

## 修正方法
行412（`const md1 = ...`の前）に以下のnullガードを追加:

```typescript
if (!listing) {
  throw new Error('Listing not found after image processing');
}
```

これにより行413以降で listing が non-null と推論される。
行413の `listing!` の `!` も不要になるので `listing` に変更する。

## 制約
- この1箇所だけ修正する
- 他のファイルは変更しない
