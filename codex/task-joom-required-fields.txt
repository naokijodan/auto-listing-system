## タスク: Joom必須3フィールド修正（Phase 0-B）

Joom出品で必須の3フィールドが未送信の問題を修正する。

### 修正1: Store ID

**方針**: 環境変数 `JOOM_STORE_ID` から読み取る

#### apps/worker/src/lib/joom-api.ts
- `createProduct()` メソッド内で `process.env.JOOM_STORE_ID` を取得
- リクエストボディのトップレベルに `store_id: storeId` を追加
- storeIdが未設定の場合はwarningログを出す（エラーにはしない）

#### .env.example
- `JOOM_STORE_ID=` を追加（空文字、ユーザーが後で設定）

### 修正2: Currency

#### apps/worker/src/lib/joom-api.ts
- `createProduct()` のリクエストボディに `currency: 'USD'` を追加（トップレベル）
- 現在バリアント内にpriceがstringで入っているが、トップレベルにも`currency`を入れる

### 修正3: Shipping Weight (kg)

#### apps/worker/src/lib/joom-publish-service.ts
- `publishToJoom()` メソッドで、productからweightを取得
- JoomProduct interfaceに `weight?: number` を追加（kg単位）
- product.weight（グラム）を kg に変換: `product.weight ? product.weight / 1000 : 0.15`
- デフォルト値: weight未設定時は0.15kg（時計のデフォルト）

#### apps/worker/src/lib/joom-api.ts
- `createProduct()` の variants[0] の `shippingWeight` を固定200gから
  `product.weight ? product.weight * 1000 : 200` に変更（バリアントはグラム単位の可能性あり）
- ただし、Joom APIの仕様でkg単位が求められる場合は `product.weight || 0.15` （kg単位）で送信

以下の流れで修正:
1. joom-publish-service.ts の JoomProduct interface に weight フィールド追加
2. publishToJoom() で product.weight を kg変換して JoomProduct に渡す
3. joom-api.ts の createProduct() で:
   - store_id を追加
   - currency: 'USD' を追加
   - shippingWeight を product.weight ベースに変更（kg単位で送信）

### 注意事項
- Joom APIのshippingWeightがkg単位かg単位かは以下で判断:
  - 計画書によると「kg単位」とあるので、kg単位で送信する
  - つまり 200g の場合は 0.2 を送信
- product.weight はDBでグラム単位（Int?）
- .env は変更しない（.env.example のみ）
- テストファイルは影響があれば修正する
- 修正後に `cd ~/Desktop/rakuda && npm test --workspace=apps/worker` で全テストが通ることを確認
