## タスク: Phase 1-E - eBayポリシー名同期

### 概要
eBayのビジネスポリシー（Fulfillment/Payment/Return）をDBに保存し、
ポリシー名でUI選択→IDで出品時に使用できるようにする。

### 実装1: Prismaスキーマ - EbayPolicyモデル追加

#### packages/database/prisma/schema.prisma に追加:

```prisma
model EbayPolicy {
  id              String    @id @default(cuid())
  type            String    // 'FULFILLMENT' | 'PAYMENT' | 'RETURN'
  policyId        String    // eBay側のポリシーID
  name            String    // ポリシー名（表示用）
  description     String?   // ポリシー説明
  marketplaceId   String    @default("EBAY_US")
  isDefault       Boolean   @default(false)
  details         Json?     // ポリシーの詳細情報（送料設定等）
  createdAt       DateTime  @default(now())
  updatedAt       DateTime  @updatedAt

  @@unique([type, policyId, marketplaceId])
  @@map("ebay_policies")
}
```

追加後:
```bash
cd ~/Desktop/rakuda && npx prisma generate --schema=packages/database/prisma/schema.prisma
```

マイグレーションは生成のみ（適用はしない。devDBが無い場合があるため）:
```bash
cd ~/Desktop/rakuda && npx prisma migrate dev --schema=packages/database/prisma/schema.prisma --name add-ebay-policy-table --create-only 2>/dev/null || true
```

### 実装2: syncPolicies()メソッド追加（ebay-api.ts）

`apps/worker/src/lib/ebay-api.ts` の EbayApiClient クラスに以下のメソッドを追加:

```typescript
/**
 * eBayのビジネスポリシーをDBに同期
 * Fulfillment/Payment/Returnの3種をまとめて取得・保存
 */
async syncPolicies(marketplaceId: string = 'EBAY_US'): Promise<{
  fulfillment: number;
  payment: number;
  return: number;
}>
```

実装:
1. getFulfillmentPolicies(marketplaceId) を呼ぶ
2. 結果の各ポリシーについて `prisma.ebayPolicy.upsert()`:
   - where: `{ type_policyId_marketplaceId: { type: 'FULFILLMENT', policyId: policy.fulfillmentPolicyId, marketplaceId } }`
   - update: `{ name: policy.name, description: policy.description, details: policy }`
   - create: `{ type: 'FULFILLMENT', policyId: policy.fulfillmentPolicyId, name: policy.name, description: policy.description, marketplaceId, details: policy }`
3. getPaymentPolicies, getReturnPolicies でも同様に実施
4. 各タイプの同期件数を返す

### 実装3: getPolicyByName()メソッド追加（ebay-api.ts）

```typescript
/**
 * ポリシー名からIDを取得（DB検索）
 */
async getPolicyByName(
  type: 'FULFILLMENT' | 'PAYMENT' | 'RETURN',
  name: string,
  marketplaceId: string = 'EBAY_US'
): Promise<string | null>
```

実装:
- `prisma.ebayPolicy.findFirst({ where: { type, name: { contains: name, mode: 'insensitive' }, marketplaceId } })`
- 見つかればpolicyIdを返す、なければnull

### 実装4: processCreateOffer()でのポリシー名解決（ebay-publish.ts）

`apps/worker/src/processors/ebay-publish.ts` の processCreateOffer() を修正:

現在は marketplaceData から直接 policyId を取得しているが、
名前が指定されている場合はDB検索でIDに変換する:

```typescript
// 既存: 直接ID
let fulfillmentPolicyId = marketplaceData.fulfillmentPolicyId;

// 追加: 名前→ID変換
if (!fulfillmentPolicyId && marketplaceData.fulfillmentPolicyName) {
  fulfillmentPolicyId = await ebayApi.getPolicyByName('FULFILLMENT', marketplaceData.fulfillmentPolicyName, marketplaceId) || undefined;
}
```

payment, return についても同様。
