## タスク: Phase 3-C - eBay複数アカウント管理

### 概要
eBay複数アカウント対応を追加。MarketplaceCredentialのcredentialIdを
出品フローに組み込み、アカウント切り替えを可能にする。

### 対象ファイル
- apps/worker/src/lib/ebay-api.ts（credentialId活用）
- apps/worker/src/processors/ebay-publish.ts（credentialId指定対応）

### 実装1: EbayApiClientの複数インスタンス管理

`apps/worker/src/lib/ebay-api.ts` のファイル末尾にある、
シングルトンのexportを修正:

現在:
```typescript
export const ebayApi = new EbayApiClient();
```

修正:
```typescript
// デフォルトインスタンス（後方互換性維持）
export const ebayApi = new EbayApiClient();

// credentialIdベースのインスタンス管理
const clientCache = new Map<string, EbayApiClient>();

/**
 * credentialIdに基づいたEbayApiClientインスタンスを取得
 * キャッシュ済みのインスタンスがあればそれを返す
 */
export function getEbayClient(credentialId?: string): EbayApiClient {
  if (!credentialId) return ebayApi;

  let client = clientCache.get(credentialId);
  if (!client) {
    client = new EbayApiClient(credentialId);
    clientCache.set(credentialId, client);
  }
  return client;
}
```

### 実装2: processCreateInventoryItem/processCreateOffer等でcredentialId対応

`apps/worker/src/processors/ebay-publish.ts` の各処理関数を修正:

1. importに追加:
```typescript
import { ebayApi, getEbayClient } from '../lib/ebay-api';
```

2. processCreateInventoryItem()の先頭:
```typescript
const marketplaceData = (listing.marketplaceData as any) || {};
const client = getEbayClient(marketplaceData.credentialId);
```

3. 以降の `ebayApi.` 呼び出しを `client.` に置換:
   - `ebayApi.ensureInventoryLocation()` → `client.ensureInventoryLocation()`
   - `ebayApi.createOrUpdateInventoryItem()` → `client.createOrUpdateInventoryItem()`

4. processCreateOffer()でも同様:
   - `ebayApi.ensureDefaultPolicies()` → `client.ensureDefaultPolicies()`
   - `ebayApi.createOffer()` → `client.createOffer()`

5. processPublishOffer()でも同様:
   - `ebayApi.publishOffer()` → `client.publishOffer()`

6. processEndListing()でも同様:
   - `ebayApi.withdrawOffer()` → `client.withdrawOffer()`

7. processSyncStatus()でも同様:
   - `ebayApi.getOffer()` → `client.getOffer()`

**注意**: 各関数内のlistingからmarketplaceData.credentialIdを取得し、getEbayClient()でインスタンスを取得する。
credentialIdが未設定の場合はデフォルトインスタンス（ebayApi）が使われるので後方互換性は維持される。

### 注意事項
- デフォルトインスタンス（`export const ebayApi`）は残す（後方互換性）
- clientCacheはMap<string, EbayApiClient>でメモリ内キャッシュ
- processの各関数でlistingのmarketplaceDataからcredentialIdを取得
- credentialIdが未指定の場合はデフォルト動作（既存と同じ）
- テストが壊れないようにする
- 修正後に `cd ~/Desktop/rakuda && npm test --workspace=apps/worker` で全テストが通ることを確認
