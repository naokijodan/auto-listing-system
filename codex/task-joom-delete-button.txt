## タスク: Joom出品ページに削除ボタンを追加

### 目的
apps/web/src/app/joom/page.tsx に各Listing行の削除ボタンと確認ダイアログを追加する。

### 前提
- DELETE API: DELETE /api/joom/listings/:id（実装済み、BullMQキュー経由で非同期削除）
- 既存UI: apps/web/src/app/joom/page.tsx（979行）
- deleteApi関数: apps/web/src/lib/api.ts（import済み？要確認）
- Trash2アイコン: lucide-reactから既にimport済み（30行目）

### 修正内容

#### 1. 削除確認ステート追加
```tsx
const [deleteTarget, setDeleteTarget] = useState<string | null>(null);
const [isDeleting, setIsDeleting] = useState(false);
```

#### 2. 削除ハンドラー追加
```tsx
const handleDelete = useCallback(async (listingId: string) => {
  setIsDeleting(true);
  try {
    await deleteApi(`/api/joom/listings/${listingId}`);
    addToast({ type: 'success', message: '削除リクエストを送信しました' });
    mutate(); // SWRを再取得
  } catch (error) {
    addToast({ type: 'error', message: '削除に失敗しました' });
  } finally {
    setIsDeleting(false);
    setDeleteTarget(null);
  }
}, [mutate]);
```

#### 3. テーブル行の操作列に削除ボタン追加
既存の操作ボタン（Pause/Play等）の隣に追加:
```tsx
<button
  onClick={() => setDeleteTarget(listing.id)}
  className="text-red-500 hover:text-red-700 p-1 rounded hover:bg-red-50 dark:hover:bg-red-900/30"
  title="削除"
>
  <Trash2 className="w-4 h-4" />
</button>
```

#### 4. 削除確認ダイアログ
deleteTargetがnull以外の時に表示するシンプルなモーダル:
- オーバーレイ（固定位置、背景暗転）
- 「この出品をJoomから削除しますか？」メッセージ
- 「キャンセル」「削除する」ボタン
- 削除中はLoader2スピナー表示

### 注意事項
- 既存コードの構造を壊さないこと
- import文に deleteApi がない場合は追加する
- 既存のTrash2 importは既にある（確認済み）
- Tailwind CSSのみ使用
- ダークモード対応（dark: プレフィックス）
