## Task: Migrate remaining 5 worker files from old joom-api.ts to new v3 clients

Migrate the following 5 files that still import from `./joom-api` to use the new v3 clients from `./joom/`.

### Important context:
- New v3 clients location: `apps/worker/src/lib/joom/`
- Available v3 clients: JoomProductsClient, JoomOrdersClient, JoomShippingDestinationsClient, JoomWarehousesClient
- Each v3 client extends JoomBaseClient and is instantiated with `new XxxClient()`
- v3 response format: `{ data: T }` for single, `{ data: { items: T[] } }` for lists (NO `.success` field)
- Old API used `{ success: boolean, data?: T, error?: { code, message } }` pattern
- The old joom-api.ts also exports: `joomApi` (singleton), `refreshJoomToken()`, `isJoomConfigured()`, `calculateJoomPrice()`
- `refreshJoomToken()` and `calculateJoomPrice()` and `isJoomConfigured()` are standalone functions that use prisma directly â€” they should remain available. Move them to a new file or keep them accessible.

### Migration strategy:
1. Create a compatibility/facade file at `apps/worker/src/lib/joom/compat.ts` that:
   - Instantiates v3 clients as singletons
   - Re-exports `refreshJoomToken`, `isJoomConfigured`, `calculateJoomPrice` (copy from old joom-api.ts)
   - Provides a `joomApi` facade object that delegates to v3 clients for backward-compatible usage

2. Update each file to import from `./joom` or `./joom/compat` instead of `./joom-api`
