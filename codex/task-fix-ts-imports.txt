## タスク: Worker TypeScriptエラー修正 - Import問題 (TS2614 + TS2305)

### 目的
`apps/worker/` の `tsc --noEmit` で発生するTypeScriptエラーを修正する。
CI (GitHub Actions) の typecheck を通すことが目標。

### 前提
- `@rakuda/database` パッケージの index.ts は `export * from '@prisma/client'` で全てのPrisma型を再エクスポートしている
- Prismaスキーマは `packages/database/prisma/schema/` 配下13ファイルに分割されている
- CIではこれらのenum importがTS2614エラーになっている

### 修正方針

#### TS2614エラーの修正
`import { X } from '@rakuda/database'` でTS2614が出る場合、型定義として使われているenum/typeを修正する。

**方法**: これらの型は `@prisma/client` に実在するが、CIのビルド順序の問題で `@rakuda/database` 経由だと解決できない場合がある。
→ **全てのenum importを `@prisma/client` からの直接importに変更する**

対象ファイル（全て `apps/worker/src/` 配下）:

- lib/audit-log-service.ts: ActorType, AuditEntityType, AuditAction, AuditSeverity

- processors/price-sync.ts: Marketplace

例:
```typescript
// 修正前
import { Marketplace } from '@rakuda/database';

// 修正後
import { Marketplace } from '@prisma/client';
// prismaインスタンスが必要な場合は別行で:
import { prisma } from '@rakuda/database';
```

注意: `prisma` インスタンスは引き続き `@rakuda/database` からインポートする。型/enumのみ `@prisma/client` に変更する。

#### TS2305エラーの修正
`@prisma/client` にも存在しない型（Prismaスキーマに未定義のモデル/enum）を修正する。

**方法**: ローカルにTypeScript interfaceまたはtype aliasを定義する。各ファイルの先頭でstubとして `type X = any` を追加する。

対象:
- lib/ai-service.ts: AiModel, PricePrediction, DemandForecast, ProductRecommendation, AiTrainingJob, AiPredictionLog, PriceOptimization, CompetitorPrice, AiModelType, AiProvider, Marketplace, PriceRecommendedAction, DemandForecastTarget, ForecastPeriod, DemandTrend, RecommendationType, AiTrainingJobType, AiTrainingStatus, AiPredictionType, AiPredictionStatus, PriceOptimizationTarget, PriceOptimizationStrategy, PriceOptimizationMetric
  - ただし Marketplace は @prisma/client に存在するのでそちらからimport
  - 残りはスキーマに存在しない → `type X = any` で定義

- lib/search-service.ts: SearchIndex, SearchLog, SearchSynonym, SearchFilter, PopularSearch, SearchSuggestion, SearchAnalyzer, SearchSyncMode, SearchSyncStatus, SearchQueryType, SynonymType, SearchFilterType, SuggestionSource

- lib/visualization-service.ts: ChartConfig, ChartData, DashboardChartItem, WidgetPreset, UserFeedback, PredictionFeedback, ChartType, ChartDataSource, DataGranularity, AggregationType, WidgetCategory, FeedbackTargetType, FeedbackType, FeedbackStatus, PredictionFeedbackType

- lib/workflow-service.ts: Workflow, WorkflowStep, WorkflowExecution, WorkflowStepExecution, ApprovalRequest, ApprovalAction, AutomationRule, AutomationExecution, WorkflowTriggerType, WorkflowTriggerSource, WorkflowStepType, WorkflowExecutionStatus, WorkflowStepStatus, ApprovalRequestType, ApprovalStatus, ApprovalActionType, ConditionLogic, AutomationExecutionStatus
  - 注意: operations.prisma にWorkflow関連モデルが存在するか再確認。存在すれば `@prisma/client` からimport

#### TS2724エラーの修正
- lib/ebay-publish-service.ts(3): `createEbayApiClient` → `EbayApiClient` に変更

### 実行手順
1. 各ファイルのimport文を修正
2. 修正後 `cd apps/worker && npx tsc --noEmit 2>&1 | grep "error TS2614\|error TS2305\|error TS2724" | wc -l` でエラー数がゼロになることを確認
3. `cd apps/worker && npx vitest run` でテストが全パスすることを確認
