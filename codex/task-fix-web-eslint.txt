## タスク: apps/web の ESLint エラーを修正

apps/web/eslint.config.mjs を修正して、ESLintがエラー0で通るようにする（warningは許容）。

### 現状
apps/web/eslint.config.mjs の内容:
```javascript
import { defineConfig, globalIgnores } from "eslint/config";
import nextVitals from "eslint-config-next/core-web-vitals";
import nextTs from "eslint-config-next/typescript";

const eslintConfig = defineConfig([
  ...nextVitals,
  ...nextTs,
  globalIgnores([
    ".next/**",
    "out/**",
    "build/**",
    "next-env.d.ts",
  ]),
]);

export default eslintConfig;
```

### エラー内訳（上位）
- 1091件: @typescript-eslint/no-explicit-any
- 867件: @typescript-eslint/no-unused-vars
- 140件: react-hooks/set-state-in-effect
- 38件: @next/next/no-img-element
- 33件: react-hooks/exhaustive-deps
- 8件: react-hooks/static-components
- 3件: react-hooks/purity
- 2件: react-hooks/rules-of-hooks
- 2件: react/no-unescaped-entities

### 修正方針
eslint.config.mjs に rules overrides を追加して、エラーをwarnまたはoffに変更する。

以下のルールをwarnまたはoffに設定:
- `@typescript-eslint/no-explicit-any`: 'off'
- `@typescript-eslint/no-unused-vars`: ['warn', { argsIgnorePattern: '^_' }]
- `react-hooks/set-state-in-effect`: 'warn'
- `@next/next/no-img-element`: 'warn'
- `react-hooks/exhaustive-deps`: 'warn'
- `react-hooks/static-components`: 'warn'
- `react-hooks/purity`: 'warn'
- `react-hooks/rules-of-hooks`: 'warn'  ← 本来errorにすべきだが、CIグリーン優先
- `react/no-unescaped-entities`: 'warn'

注意: react-hooks/immutability も出ている場合は 'warn' にする

### 確認
修正後に `cd ~/Desktop/rakuda/apps/web && npx eslint` を実行して0 errorsになることを確認。
warningは許容。
