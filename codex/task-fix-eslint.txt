## タスク: ESLint設定ファイル追加（5パッケージ）

ESLint v9.39.2がインストール済み。`typescript-eslint` もルートの node_modules にある。
以下の5パッケージにlintスクリプトがあるが、eslint.config.mjsが存在しない。

### 対象パッケージ
1. apps/api
2. apps/worker
3. packages/config
4. packages/logger
5. packages/schema

### 各パッケージに作成するファイル: eslint.config.mjs

TypeScriptプロジェクト用の最小限の設定を作成する。

```javascript
import eslint from '@eslint/js';
import tseslint from 'typescript-eslint';

export default tseslint.config(
  eslint.configs.recommended,
  ...tseslint.configs.recommended,
  {
    ignores: ['dist/**', 'build/**', 'node_modules/**', '**/*.js', '**/*.mjs'],
  },
  {
    rules: {
      '@typescript-eslint/no-explicit-any': 'off',
      '@typescript-eslint/no-unused-vars': ['warn', { argsIgnorePattern: '^_' }],
    },
  },
);
```

### 注意事項
- 5つのパッケージすべてに同じ内容の `eslint.config.mjs` を作成する
- apps/web は既に独自のeslint.config.mjsがあるので変更しないこと
- 作成後に `cd ~/Desktop/rakuda && npm run lint` を実行して成功することを確認
- エラーが出た場合は、warnレベルに落とすかルールをoffにして対応（CIグリーンが目的）
- `@eslint/js` パッケージがなければ `npm install -D @eslint/js` を実行（ルートで）
